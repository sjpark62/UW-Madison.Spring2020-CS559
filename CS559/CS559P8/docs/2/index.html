Box 1: Euler Angles

We use the term “Euler Angles” to refer to the representation of a 3D rotation as a sequence of 3 rotations about fixed axes. For example, we can represent a rotation in 3D as a sequence of 3 rotations: one about the X axis, one about the Y axis, and one about the Z axis. The important point is these axes are fixed: we decide them ahead of time. We have chosen X, Y then Z. Once we’ve chosen them, we need to stick with these 3 axes in this order.

This representation comes from a theorem that mathematician Leonhard Euler had: any rotation in 3D can be represented as a sequence of rotations about fixed axes. The theorem is very general: any set of axes can work (providing no two consecutive axes are parallel).

Once we choose a convention (a set of axes and their order), we only need 3 numbers to represent an arbitrary 3D rotation: the amount of rotation around each axis.

If only one of the axes is non-zero, things are easy. Experiment with the Demo (below) and make sure you understand what a rotation about each axis does.

A thing that makes Euler Angles tricky is that the different rotations interact. The first rotation changes the coordinate system (frame) that the next rotation takes place in, so we need to interpret each of the angles with respect to the others.

Experiment with rotating about one axis and then another. For example, change X1 and then Y1. Then reset (press the zero button) and see what happens if you change Y1 then X1. Try to understand what happens as we apply a sequence of rotations.

Here are some complicated situations that come up:

  1. In some circumstances, you can get into a configuration where there are only two possible directions to rotate (you can cause a particular change to happen). This is called gimbal lock. For XYZ Euler angles this occurs when the rotation about the Y axis is 90 degrees, causing the X axis to align with the Z axis (and perform the same motion).

    To see this in the demo, press the “Gimball Lock” button which will set Y to 90 degrees. Notice how X and Z do the same thing.

  2. Different sets of angles may represent the same rotation. That means that interpolating between two close things might lead to weird paths, or that comparing sets of angles is hard.

    To see an example of this, press the “Bad Interp” Button. You’ll see that the rotations are the same (the objects are in the same configuration), but the numbers are very different. If you interpolate between the two configurations that should be “the same”, you get a weird rotation.

  3. Different orders do different things.

    To try this out, set object #2 to ZYX (with object #1 as XYZ). Notice how they do different things. Start with just changing X and Y of each. See how the rotations “closer to the object” are object axes, while rotations “closer to the world” are world axes.

  4. In general, it can be hard to know what changes to the different rotations will do.

To demonstrate these things there’s an interactive toy on the bottom of this page. You are welcome to read the code, but it’s not recommend. You should play with the toy to see how Euler angles work.

In the view, you’ll see three objects. For the ones on the left and right, there is a choice drop down to let you choose the axes and their order. Then you can use the sliders to adjust the angles and see what happens. There are buttons that put things into gimbal lock, and another one that creates two rotations that are the same rotations, but with very different parameters - interpolation between the two yields a crazy path. The center object will interpolate between the left and right (but only if they are using the same axes and order).

We recommend that you spend some time playing with the Euler Angle toy (below and 2-1.js (2-1.html)) to build up your intuitions about how Euler Angles work, and the problems they have.

Then you can move on to Page3(Comparing Incremental Rotations) to consider some alternatives.
