Polygon Filling

In class, we discussed that there were different rules for filling polygons, and that HTML 2D Canvas supported multiple ones. We went over the rules briefly in class.

The official documentation for fill says very little about the difference between the two rules (“non-zero” and “even-odd”). It defers to Wikipedia to explain non-zero and even-odd.

The HTML Living Standard has a nice concise explanation:

  • The value “nonzero” value indicates the nonzero winding rule, wherein a point is considered to be outside a shape if the number of times a half-infinite straight line drawn from that point crosses the shape's path going in one direction is equal to the number of times it crosses the path going in the other direction.

  • The “evenodd” value indicates the even-odd rule, wherein a point is considered to be outside a shape if the number of times a half-infinite straight line drawn from that point crosses the shape's path is even.

In box 1-1, (1-1.html and 1-1.js), practice using these two different fill rules: draw the same path twice, such that you get a different picture when you draw it with the different rules. Your path should include some curved segments. Please do not use the example I showed in lecture. Use a translation to make the paths not be on top of each other.
